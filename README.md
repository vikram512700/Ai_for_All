## ✨ What Is This?

**AI for All** is a self-contained, offline-capable learning portal built for **Cloud Engineers, DevOps Engineers, and Software Developers** who want to transition into AI Engineering — without a PhD, without expensive bootcamps, and without wading through scattered YouTube tutorials.

Every module is a rich, interactive HTML page with:

- 🎨 **Visual diagrams** — custom CSS architecture visualizations, no static images needed
- 📖 **Deep explanations** — production-level context, not toy examples
- 💻 **Syntax-highlighted code** — Python snippets with keyword coloring
- ❓ **Interview Q&A** — accordion-style, 3–5 questions per module
- 🔗 **Cross-module links** — every concept connects to where it's explored next

## 🚀 Getting Started

### Option A — Just Open It (Recommended)

```bash
# 1. Clone the repo
git clone https://github.com/yourname/ai-for-all.git
cd ai-for-all

# 2. Open in your browser — no server needed
open index.html          # macOS
start index.html         # Windows
xdg-open index.html      # Linux
```

### Option B — Local Dev Server (for live reload)

```bash
# Python (no install needed)
python -m http.server 8080
# → Visit http://localhost:8080

# Node.js
npx serve .
# → Visit http://localhost:3000

# VS Code
# Install "Live Server" extension
# Right-click index.html → "Open with Live Server"
```

## 🤝 Contributing

Contributions are welcome. Here's how the codebase works so you can hit the ground running:

**Architecture rules:**
1. **Each HTML page is self-contained** — its `<style>` block holds all page-scoped CSS
2. **Global styles only** in `styles.css` — don't duplicate what's already there
3. **Sidebar HTML is repeated** across all 22 pages — add nav items to every file
4. **Icons** use Ionicons 7.1.0 via `<ion-icon name="...">` — browse at [ionicons.com](https://ionic.io/ionicons)
5. **Diagrams are pure HTML/CSS** — no image uploads, no Mermaid (except `architecture-patterns.html`)
6. **Colors** always use CSS custom properties like `var(--accent-purple)` — never hardcoded hex in shared CSS

**PR checklist:**
```
☐ Renders correctly in dark mode (default) and light mode
☐ Mobile layout works at 375px viewport width
☐ Active nav item highlights correctly (driven by script.js)
☐ No hardcoded colors in shared CSS — use var(--accent-*)
☐ No new external JS dependencies
☐ At least one visual diagram per new module
☐ At least 3 interview Q&A accordion items
☐ Sidebar nav entry added to ALL html files
```
