Module 15

AI Architecture Patterns

The definitive visual guide to the 7 core design patterns used in modern AI engineering.

Pattern 1: Basic Chatbot

The simplest form of AI interaction. Stateless passing of prompts to an LLM API.

graph TD User --> LLM[LLM API] LLM --> Response

Pattern 2: RAG Architecture

Retrieval-Augmented Generation to ground LLMs in private data.

graph TD User --> Query Query --> Embed[Embedding Model] Embed --> VectorSearch[(Vector DB)] VectorSearch --> Context[Retrieved Chunks] Query --> Prompt Context --> Prompt Prompt --> LLM LLM --> Response

Pattern 3: Agent Engineering

An LLM equipped with tools and a loop to reason and act.

graph TD User --> Agent Agent <--> Tools Tools <--> APIs[External APIs] Agent --> Response

Pattern 4: MCP Architecture

Standardized secure local tool connection via Model Context Protocol.

graph LR LLM --> MCP_Client[MCP Client] MCP_Client <--> MCP_Server[MCP Server] MCP_Server --> Tool[Local Tools/DB]

Pattern 5: Multi-Agent System

Specialized agents routed by a central supervisor.

graph TD Supervisor --> Research[Research Agent] Supervisor --> Coding[Coding Agent] Supervisor --> Review[Review Agent] Research --> Supervisor Coding --> Supervisor Review --> Supervisor

Pattern 6: Human-In-The-Loop

Pausing execution for human approval before dangerous actions.

graph LR Agent --> Approval{Human Approval} Approval -- "Approve" --> Execution[Execute Action] Approval -- "Reject" --> Agent

Pattern 7: AI Gateway Pattern

Load balancing, routing, and fallbacks for enterprise API resilience.

graph TD User --> Gateway{AI Gateway / LiteLLM} Gateway -- "High Intelligence" --> GPT4[GPT-4o API] Gateway -- "Fast / Cheap" --> Haiku[Claude 3 Haiku] Gateway -- "Open Source" --> Local[Ollama / DeepSeek] Gateway -- "Fallback" --> Gemini[Gemini Pro]
