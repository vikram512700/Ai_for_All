AI & LLM Fundamentals

Section 01

What is a Large Language Model?

A Large Language Model (LLM) is a neural network — specifically a Transformer — trained on hundreds of billions of words scraped from the internet, books, and code. Its core task is deceptively simple: predict the most likely next token given all previous tokens. Everything else — reasoning, coding, translation, summarization — emerges from doing this extremely well at scale.

The Autocomplete Analogy

An LLM is your phone's autocomplete — scaled up by a factor of a trillion. When you type "The capital of France is…", your phone might suggest "Paris". An LLM does this across multi-paragraph reasoning chains, code blocks, and logical deductions simultaneously.

How It Learns

During pre-training, the model reads trillions of tokens and adjusts billions of internal parameters (weights) so its next-token predictions improve. No human labels needed — the text itself is the training signal. This is called self-supervised learning.

Emergent Capabilities

Nobody programmed an LLM to write poetry or debug code — these abilities emerged from scale. Once models cross a certain parameter threshold (~7B+), they start showing abilities that smaller models completely lack. This is called emergence.

Key Insight: LLMs are not databases retrieving stored facts. They are probability machines generating the statistically likely continuation of your prompt. This is why they can be wrong — and why RAG (connecting them to real data) is so important.

Section 02

Tokens — The Atoms of Language

LLMs don't read words — they read tokens. A token is a chunk of text: sometimes a full word, sometimes a word-piece, sometimes a single character or punctuation mark. The process of splitting text into tokens is called tokenization.

Cost & Latency: API pricing is always per-token (input + output). A typical GPT-4o call costs ~$5 per million input tokens. Knowing your token count before calling an LLM is a critical FinOps skill.

Vocabulary Size

Modern LLMs use a tokenizer with a fixed vocabulary — typically 50,000–100,000 tokens. GPT-4 uses cl100k_base (100,000 tokens). Claude uses its own tokenizer. Any string can be encoded using this vocabulary.

Code Tokenization

Code tokens are often different — print is one token, def is one token, but unusual variable names like myVar_123 might be split into 3–4 tokens. This affects how efficiently models read code.

Section 03

Context Window — The Model's Working Memory

The context window is the maximum number of tokens an LLM can process in a single request — both your input (prompt) and its output (completion) combined. Everything outside the context window is invisible to the model.

What Fits in 128K?

128K tokens ≈ ~300 pages of a book, ~10,000 lines of code, or a full research paper with all its references. This is why modern models can summarize entire codebases in one call.

Lost in the Middle

Research shows LLMs perform best on information at the beginning and end of the context window, and worst on content buried in the middle. For RAG, put the most relevant chunks first or last.

Cost Scales with Tokens

Sending a 200K token context on every API call is expensive. Good AI engineering means designing smart context management — only sending what the model actually needs for the current task.

Section 04

Temperature & Sampling Parameters

After the model produces a probability distribution over the next token, sampling parameters decide how to pick from that distribution. These are the levers that control creativity vs. precision.

0.0

Deterministic

Always picks the single most probable token. No randomness. Same prompt → same output every time.

Use for: structured data extraction, SQL generation, JSON formatting
0.7

Balanced

The default for most applications. Introduces variation while staying coherent and on-topic.

Use for: chatbots, Q&A, summarization, code generation
1.5

Creative

High randomness. Outputs vary widely. Can produce brilliant or nonsensical results.

Use for: brainstorming, creative writing, generating diverse ideas

Top-P (Nucleus Sampling)

Instead of considering all tokens, only sample from the smallest set of tokens whose cumulative probability exceeds P. top_p=0.9 means: only pick from the top 90% probability mass. Keeps outputs sensible while allowing variety.

Top-K

Only sample from the top K most probable tokens at each step. top_k=40 means the model can only pick from the 40 highest-probability words. A hard filter, as opposed to Top-P's soft probability cutoff.

Max Tokens (Output Limit)

Hard cap on how many tokens the model generates in its response. Essential for cost control — if you don't set it, the model may generate until it hits the context window limit, costing you money.

Best Practice: For production applications, set temperature=0 and max_tokens explicitly. Only increase temperature for creative tasks. Never use high temperature for factual or structured output tasks.

Section 05

The Transformer Architecture (Simplified)

All modern LLMs are built on the Transformer architecture, introduced by Google in 2017 ("Attention Is All You Need"). You don't need to implement one — but understanding the key components helps you reason about model behaviour.

1. Input Tokenization

Text is split into tokens and each token is mapped to a unique integer ID.

▼

2. Embeddings + Positional Encoding

Each token ID becomes a high-dimensional vector. Position info (order) is added so the model knows which token came first.

▼

3. Self-Attention Layers (×N)

The core of the Transformer. Each token "attends" to every other token and learns which ones are relevant. GPT-4 has ~96 of these layers stacked.

▼

4. Feed-Forward Networks

After attention, each token passes through a large feed-forward network that stores factual knowledge in its weights. This is where the model's "memory" lives.

▼

5. Output: Next-Token Probability Distribution

A final layer (the "language model head") converts the last hidden state into probabilities over the entire vocabulary. The model picks the next token and repeats.

Self-Attention in plain English: When reading the word "it" in "The animal didn't cross the street because it was too tired", self-attention lets the model figure out that "it" refers to "animal" — by weighing how much each prior word should influence the representation of "it".

Embeddings — Meaning as Numbers

An embedding is a dense numerical vector (an array of floating-point numbers) that represents the semantic meaning of a piece of text. Words or sentences with similar meanings have embeddings that are close together in vector space.

Vector Arithmetic

The famous equation: King − Man + Woman ≈ Queen. Embedding spaces encode relationships. You can add and subtract concepts and navigate the space mathematically. This is the foundation of semantic search.

Semantic Search

Instead of keyword matching, semantic search embeds your query into a vector and finds documents whose vectors are closest by cosine similarity. "What causes rain?" finds "precipitation mechanisms" even without the word "rain".

Where They're Stored

Embeddings are stored in Vector Databases like Pinecone, Qdrant, pgvector, or Weaviate. These databases are optimised for Approximate Nearest Neighbour (ANN) search — finding similar vectors in milliseconds across millions of documents.

Popular Embedding Models: text-embedding-3-small (OpenAI, 1536 dims), text-embedding-3-large (3072 dims), embed-english-v3 (Cohere), nomic-embed-text (open-source, runs locally via Ollama).

Section 07

Hallucination — Why LLMs Make Things Up

A hallucination is when an LLM confidently generates text that is factually incorrect. The model doesn't "know" it's wrong — it's just producing a statistically plausible continuation of your prompt.

Why It Happens

The model's goal is to generate a plausible next token, not a true one. When asked about obscure facts it wasn't trained on, it fills gaps with plausible-sounding fiction. It has no mechanism to say "I don't know."

Common Triggers

Asking for specific dates, citations, code API signatures, medical dosages, or legal statutes. Any domain where precise, verifiable facts matter and the model's training data may be sparse or outdated.

The Fix: RAG

Retrieval-Augmented Generation (RAG) grounds the model in real documents at runtime. Instead of relying on trained weights, the model reads your actual data first — then answers. This is why RAG is the #1 enterprise AI pattern.

Section 09

Training vs Inference vs Fine-Tuning

Pre-Training

Training a model from scratch on billions of tokens. Costs $10M–$100M+ in GPU compute. Done by OpenAI, Anthropic, Meta, Google. You almost never do this as an AI Engineer.

Very ExpensiveMonths of GPU time

Fine-Tuning

Taking a pre-trained model and continuing training on your specific domain data. Makes the model better at your use case. Costs hundreds to thousands of dollars. Tools: LoRA, QLoRA, Axolotl, Hugging Face TRL.

Moderate CostDomain Specialization

Inference (What You Do Daily)

Sending a prompt to an already-trained model and getting a response. This is what 99% of AI Engineering involves. Pay-per-token via API, or run locally with Ollama/vLLM. No GPU training needed.

CheapInstant Results

Prompt Engineering vs Fine-Tuning

Always try prompt engineering first. A well-crafted system prompt + few-shot examples can outperform fine-tuning for many tasks — and costs nothing upfront. Fine-tune only when you have 1000+ labeled examples and prompting has plateaued.

Prompting = FastFine-tune = Expensive
Section 10

How Text Generation Works — Token by Token

LLMs don't generate an entire response at once. They generate one token at a time, each token conditioned on all previous tokens. This is why you see text appear progressively ("streaming") in ChatGPT and Claude.

The transformer architecture was originally designed for machine translation tasks, but its ability to model long-range dependencies made it ideal for language modeling at scale.

Why Streaming Matters

Without streaming, the user waits for the full response (could be 30+ seconds). With streaming via Server-Sent Events (SSE), tokens arrive as they're generated. This is the default in all modern AI APIs and critical for UX.

TTFT vs Throughput

Time-to-First-Token (TTFT) is how long until the first token appears. Throughput is tokens-per-second thereafter. For real-time chat, TTFT matters most. For batch processing, throughput matters most. Both are LLMOps metrics.

Section 11

Interview Q&A — Test Your Knowledge

Click each question to reveal the answer. These are real questions asked in AI Engineer and ML Platform Engineer interviews.

Why does the model give different outputs for the same prompt? +
Because temperature > 0 introduces probability sampling. At each step, instead of always choosing the most probable token, the model samples from a probability distribution — so different random samples produce different outputs. Set temperature=0 for fully deterministic outputs.
What is the difference between a token and a word? +
A token is the atomic unit a model processes — which may be a whole word, a subword piece, punctuation, or whitespace. On average, 1 word ≈ 1.3 tokens in English. Rare or compound words like "unbelievable" may be split into ["un", "believ", "able"] — 3 tokens. API pricing is always per token, not per word.
What happens when you exceed the context window? +
The API throws an error (context length exceeded), or with some models, older tokens are silently truncated. The model has zero memory of anything outside its context window — it cannot "remember" previous conversations unless you explicitly include them in the new prompt or use an external memory store.
Why do LLMs hallucinate and how do you prevent it? +
LLMs hallucinate because they are trained to produce plausible text, not verified truth. When asked about facts outside their training data, they generate statistically likely continuations that may be false. Prevention strategies: (1) RAG — provide real source documents at runtime, (2) set temperature=0 for factual tasks, (3) instruct the model to say "I don't know" in the system prompt, (4) use evaluation pipelines (RAGAS) to measure hallucination rates.
What is the difference between Top-P and Temperature? +
Temperature scales the entire probability distribution before sampling — high temperature flattens it (more randomness), low temperature sharpens it (more deterministic). Top-P then filters the distribution to only the tokens that cumulatively account for P% of the probability mass. They work together: temperature reshapes the distribution, Top-P then truncates it. Most APIs let you set both; a common default is temperature=0.7, top_p=0.9.
When should you fine-tune instead of prompt engineer? +
Fine-tune only when: (1) you have 1,000+ high-quality labeled examples, (2) prompt engineering has genuinely plateaued, (3) you need specific output formats or domain jargon the base model doesn't handle, or (4) you need to reduce prompt length and cost at scale. Always exhaust prompt engineering (including few-shot examples and chain-of-thought) before committing to the cost and complexity of fine-tuning.
What is RLHF and why does it matter? +
Reinforcement Learning from Human Feedback (RLHF) is the training step that turns a base language model into a helpful, safe assistant. Human raters rank model responses; a "reward model" learns to predict human preferences; the LLM is then fine-tuned using RL (PPO) to maximize that reward. This is why ChatGPT and Claude feel like assistants rather than raw text completers. A newer variant, DPO (Direct Preference Optimization), achieves similar results without the complex RL loop.
What is the difference between open-weights and open-source models? +
Open weights means the trained model parameters are publicly downloadable, but the training code and data may not be (e.g., Llama 3 — Meta releases the weights but not the full training pipeline). Open source means weights + training code + data are all publicly available (e.g., DeepSeek-R1). For AI Engineering, the key practical difference is that open-weights models can be run locally (via Ollama, vLLM, etc.) for privacy, customisation, and cost reasons.
