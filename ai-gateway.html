The Central Control Plane

What Is an AI Gateway?

An AI Gateway sits between your application and all LLM providers. Instead of each service having its own API keys, rate limiting, and cost controls, the gateway centralizes all of this — acting as a unified proxy that adds auth, caching, routing, observability, and policy enforcement to every LLM call without changing application code.

Core Gateway Features

Intelligent Model Routing

Route simple queries to cheap/fast models (Haiku, GPT-4o-mini), complex queries to powerful models (Opus, GPT-4o). Route based on request content, user tier, latency budget, or real-time provider pricing.

Semantic Caching

Cache LLM responses by semantic similarity — not exact text match. "What is the capital of France?" and "France's capital city?" both hit the same cache entry. Typical cache hit rate: 40–60%, directly reducing costs by the same percentage.

Automatic Failover

If Anthropic is down or rate-limiting, automatically retry on Azure OpenAI or Google Gemini. Applications see zero downtime. Configure fallback priority lists and failover conditions per endpoint or per model.

Budget Enforcement

Set monthly/daily cost budgets per user, team, or project. When a budget is hit: auto-downgrade to a cheaper model, queue requests, or return a budget-exceeded error. Prevents runaway agent costs from impacting the whole platform.

Input/Output Guardrails

Apply content filters, PII detection, and prompt injection screening to every request before it reaches the LLM. Check outputs for toxicity, hallucination markers, or sensitive data before returning to the user. All enforced centrally.

Unified Auth & API Keys

Application services hold one gateway API key. The gateway holds all provider API keys in a central secrets store. Rotate provider keys without touching application code. Scope key permissions to specific models or endpoints.

AI Gateway Tools: What's Available

LiteLLM
Open Source

Most popular open-source gateway. 100+ model providers, semantic cache, virtual keys, budget tracking, Prometheus metrics. Can be self-hosted or use managed LiteLLM Cloud.

Portkey
SaaS + OSS

Production-grade gateway with advanced routing rules, guardrails, and detailed analytics. Strong UI for managing routing configs. Both managed and self-hosted options.

Kong AI Gateway
Enterprise

Enterprise-grade, extends Kong's existing API gateway with LLM-specific plugins. Best for organizations already using Kong for general API management who want to unify LLM traffic.

AWS Bedrock Gateway
Cloud Native

Built into AWS Bedrock. IAM-based auth, CloudWatch metrics, VPC isolation. Best for AWS-first organizations who want zero infrastructure to manage and tight AWS service integration.

Interview Prep

Top Interview Questions

Why use an AI gateway instead of calling LLM APIs directly?
Five reasons: (1) Reliability — without a gateway, if Anthropic has an outage, your entire product goes down. A gateway automatically fails over to Azure OpenAI or Gemini in milliseconds, with zero application code changes. (2) Cost control — without centralized budget enforcement, a single runaway agent loop can generate thousands of dollars in unexpected API costs before anyone notices. Gateways can hard-cap spending per user, team, or project. (3) Security — provider API keys are kept in one secured secrets store; applications use scoped gateway tokens. Rotating or revoking a provider key happens in one place. (4) Observability — all LLM traffic flows through one point, making it trivial to track costs, latency, and errors across all models and providers. (5) Semantic caching — responses to semantically similar queries are reused, typically cutting costs 40-60% without any application changes.
How does semantic caching differ from exact-match caching?
Exact-match caching: only returns a cached response if the query string is character-for-character identical to a previous query. Hit rate for LLM queries is typically under 5% since natural language varies constantly. Semantic caching: embed the incoming query into a vector, retrieve the most similar cached vector, and if cosine similarity exceeds a threshold (typically 0.92-0.95), return the cached response. "What's the capital of France?", "France capital?", and "Where is Paris located in terms of being France's capital?" all hit the same cache entry. Hit rates of 40-60% are typical for FAQ-style applications. Implementation: the gateway hashes the query embedding as the cache key; Redis or a vector DB stores (embedding, response) pairs with a configurable TTL.
How would you implement per-user cost budgets through a gateway?
Implementation in LiteLLM (or similar gateway): (1) Create a virtual API key per user or team with a `max_budget` parameter (e.g., $10/month). (2) The gateway tracks token consumption per virtual key and converts to dollar cost using the model's price-per-token. (3) When a user hits 80% of their budget, the gateway can: emit a warning event, downgrade their requests to a cheaper model, or start returning 429 errors with a budget-exceeded message. (4) Budget resets are configurable — daily, weekly, or monthly. (5) Admins get a dashboard showing current spend per key/team. The key advantage: budget enforcement is completely transparent to application code — developers just call the gateway like any other LLM API, and the gateway handles all budget accounting in the middleware layer.
