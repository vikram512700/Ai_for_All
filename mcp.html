Model Context Protocol (MCP)

An open standard that enables secure, two-way connections between AI models and local or remote data sources, standardizing how AI tools are built and consumed.

1. What is MCP?

The Model Context Protocol is an open-source standard introduced by Anthropic. It standardizes how AI applications (Clients) interact with external data sources and tools (Servers). Instead of writing custom API wrappers for every LLM and every tool, MCP provides a unified JSON-RPC protocol.

Problem it solves:

Before MCP, if you wanted ChatGPT, Claude, and your custom LangChain agent to all read your GitHub repos, you had to write 3 different integrations. With MCP, you write one MCP Server for GitHub, and any MCP-compliant client can instantly use it.

2. Real World Analogy

Hardware Analogy (USB)

MCP is like the USB-C standard for AI. Before USB, every phone and laptop had a proprietary charger. USB-C standardized it. MCP standardizes the "plug" between AI models and data sources.

Cloud Analogy (LSP)

It is identical in philosophy to the Language Server Protocol (LSP) which standardized how IDEs (VSCode, IntelliJ) communicate with language linters (Python, TypeScript).

3. MCP Client-Server Architecture

The protocol communicates over Stdio (local command execution) or SSE (Server-Sent Events for remote connections).

4. How MCP Exposes Data

An MCP server exposes three main primitives to the AI Client:

  1. Resources: Static data that the AI can read. Like file contents, database schemas, or API documentation. The client requests a URI (e.g., file:///logs/app.log) and reads it.
  2. Tools: Executable functions the AI can call. The Server lists available tools (e.g., query_database) and their expected JSON schema. The LLM generates the JSON, and the Server executes it.
  3. Prompts: Reusable prompt templates provided by the server. The client can ask the server for a specific prompt template, fill in the variables, and send it to the LLM.

5. Hands-On Lab: Build an MCP Server

A simple Python MCP Server that exposes a tool to fetch weather data, using the mcp python SDK.

from mcp.server.fastmcp import FastMCP

# Initialize the server
mcp = FastMCP("Weather Server")

# Expose a tool to the LLM
@mcp.tool()
def get_weather(city: str) -> str:
    """Get the current weather for a specific city."""
    # Mock implementation
    weather_data = {"London": "Rainy, 12C", "Tokyo": "Sunny, 22C"}
    return weather_data.get(city, "Unknown city")

if __name__ == "__main__":
    # Start the server listening on Stdio
    mcp.run()
